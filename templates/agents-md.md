# TaskLab

> Generated by `tasklab instructions v{{VERSION}}`. Re-run to update.

TaskLab is a HITL task runner for service integrations. It always fetches the latest
curated tasks from TaskHub before running. Local tasks live in `~/.tasklab/tasks/` and
are shared across all your projects on this machine.

## Discover and run tasks

```bash
tasklab list                              # list TaskHub tasks + your local tasks
tasklab run <service/task-name>           # run a task (syncs TaskHub first)
tasklab run <task> --project-root <dir>   # specify where runtime artifacts go
```

## Git workflow rules

- **Always work on a named feature branch** — never commit directly to `main`
- **Keep commits under 200 lines** — if your staged diff exceeds 200 lines, pause and consider splitting into smaller, focused commits before proceeding. Use `git diff --staged --numstat` to check. Smaller commits are easier to review and revert.
- Branch naming: `feat/`, `fix/`, `docs/`, `chore/`

## TaskLab rules — follow these exactly

1. **Use `tasklab run`** — never call scripts under `outputs/scripts/` directly
2. **Use `tasklab list`** to discover available tasks before running
3. **Pause at HITL steps** — scripts will prompt for human action; wait before continuing
4. **Fail-closed** — if a script exits non-zero, stop and report; do not skip or proceed
5. **Secrets** — never echo, log, or commit `.env` values; reference file paths only
6. **Two-directory model**:
   - Task files live in `~/.tasklab/tasks/` (scripts, templates — no secrets, shared across projects)
   - All runtime output (`.env`, credentials, generated code) goes to `--project-root`
7. **Local overrides TaskHub** — if a task exists in `~/.tasklab/tasks/`, that version runs

## Create a new task

```bash
tasklab init                        # write AGENTS.md + .gitignore in this project
tasklab init <service/task-name>    # scaffold a new task into ~/.tasklab/tasks/
```

Edit `task.yaml` and `outputs/scripts/` in the scaffolded folder, then run it.

## Environment files

- Runtime `.env` → `--project-root` directory (gitignored there)
- Task folder → `inputs.example.yaml` only (no live values, no secrets)
- `01-preflight.sh` validates all required env vars — run it before main scripts

## Task structure

```
~/.tasklab/tasks/<service>/<task-name>/
  task.yaml              # goal, scope, inputs, outputs
  plan.yaml              # ordered steps (human-readable)
  outputs/scripts/       # numbered shell scripts
    00-hitl-links.sh     # deep links + copy-once guidance
    01-preflight.sh      # env validation (fail-closed)
    02-*.sh … 09-*.sh    # main setup steps
    99-run-tests.sh      # smoke tests
  hitl/*.step.yaml       # manual steps (dashboard / web UI)
  inputs.example.yaml    # template for operator .env values
```
